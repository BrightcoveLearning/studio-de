Einführung

Mit der Social API können Sie den Social Sharing-Status und den Verlauf für Ihre Videos abrufen. Es soll hauptsächlich bei der CMS-Integration helfen, damit Ihre Benutzer diese Informationen in Ihrem eigenen CMS-System anzeigen können.

Ausführliche Informationen zu Anfragen finden Sie in der API-Referenz.

Basis-URL

Die Basis-URL für die API lautet:

  https://edge.social.api.brightcove.com/v1

Authentifizierung

Die Authentifizierung für Anfragen erfordert eine Autorisierungs-Header :

        Authorization: Bearer {access_token}

Die access_token ist ein temporäres OAuth2-Zugriffstoken, das von der Brightcove-OAuth Service. Ausführliche Informationen zum Abrufen von Client-Anmeldeinformationen und zum Abrufen von Zugriffstoken finden Sie unter Brightcove OAuth Übersicht Thema.

Operationen

Wenn Sie Client-Anmeldeinformationen anfordern, müssen Sie die Art des Kontozugriffs oder der gewünschten Vorgänge angeben. Im Folgenden finden Sie eine Liste der derzeit unterstützten Operationen für die Soziale API :

  • video-cloud/social/status/read

Beachten Sie, dass während sich die Social API in der Beta befindet, Berechtigungen dafür vorhanden sind nicht verfügbar im Abschnitt Studio Admin zum Verwalten von API-Anmeldeinformationen. Stattdessen müssen Sie die OAuth-API verwenden, um Ihre Anmeldeinformationen zu erstellen. Brightcove Learning Services hat jedoch eine einfaches Online-Tool dafür.

Zusammenfassung der Anfragen

Nur in der aktuellen Version der API GET Anfragen werden unterstützt. Es gibt zwei Gruppen von Anforderungen, die in den folgenden Abschnitten zusammengefasst sind.

Statusanfragen

Statusanfragen Gibt den aktuellen Social-Sharing-Status für Videos zurück. Sie können den Status für alle Videos in Ihrem Konto oder für ein bestimmtes Video abrufen.

Verlaufsanfragen

Verlaufsanfragen Geben Sie den Social-Sharing-Verlauf für Videos zurück. Sie können den Verlauf aller Videos in Ihrem Konto oder eines bestimmten Videos abrufen.
