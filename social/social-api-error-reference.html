Social-Sharing-Fehler

Social-Sharing-Fehler werden nicht als separate Fehlerantwort auf die API-Anforderung zurückgegeben, sondern in einer errors Array in der normalen Antwort.

Wenn der Freigabevorgang fehlgeschlagen ist, meldet die API "result": "ERROR" und geben Sie ein Fehlerobjekt zurück, das Folgendes enthält:

  • error_code- Der vom Server zurückgegebene Fehlercode, falls vorhanden, von der sozialen Plattform aufgezeichnet.
  • error_message- Die vom Server aufgezeichnete oder gegebenenfalls von der sozialen Plattform zurückgegebene Fehlermeldung.
  • platform_error- JSON-Darstellung des von der sozialen Plattform zurückgegebenen Fehlerobjekts, falls vorhanden.

Hier ist ein Beispiel

"errors": [
  {
    "error_code": "UPLOAD_FAILURE",
    "message": "Upload error: Received Facebook error response of type OAuthException: (#100) sponsor_id requires publishing profile/page to be verified or branding partner. (code 100, subcode null)"
  }
]
