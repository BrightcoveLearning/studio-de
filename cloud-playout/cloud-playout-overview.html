Einführung

Brightcove Cloud Playout ist ein Studio-Modul, mit dem Sie eine geplante Wiedergabe von Elementen aus Ihrem VOD-Katalog in Kombination mit Live-Feeds programmieren und als linearen Kanal streamen können - alles mit der Zuverlässigkeit und Skalierbarkeit von Brightcove Live.

Cloud Playout bietet eine leistungsstarke End-to-End-Lösung für Kanäle mit geplanter Programmierung, einschließlich:

  • 24x7 lineare Kanäle
  • Popup / Virtuelle Kanäle
  • Simulierte Live-Events

Cloud Playout hilft Ihnen dabei, das Engagement des Publikums zu steigern und neue Umsatzmöglichkeiten zu schaffen, indem Sie ein sendungsähnliches Live-TV-Erlebnis nutzen, bei dem zuvor aufgezeichnete Inhalte genutzt werden. Verwenden Sie vorhandene VOD-Assets, um geplante Live-Kanäle zu erstellen.

Eigenschaften

Zu den Funktionen von Cloud Playout gehören:

  • Importieren Sie VOD-Assets aus Ihrer Video Cloud Media-Bibliothek.
  • Wechseln Sie zwischen VOD-Assets und geplanten Live-Feeds.
  • Reagieren Sie in Echtzeit auf Änderungen des Last-Minute-Zeitplans auf einfache, schnelle und zuverlässige Weise.
  • Monetarisieren Sie Ihren Kanalinhalt mit serverseitiger Anzeigeneinfügung (SSAI).
  • Konfigurieren Sie Ihren Kanal so, dass er Ihr Programm wiederholt.
  • Optimieren Sie die Wiedergabe nach Gerät.
  • Ziehen und Ablegen, um den Programmplan jederzeit neu zu ordnen, auch im laufenden Zustand.
  • Schieben Sie einen Kanal über Brightcove Live, einen Amazon S3-Bucket oder ein Ziel von Drittanbietern über RTMP-Ausgang zu Ihren eigenen und betriebenen Immobilien.
  • API zum Abrufen des EPG (Electronic Program Guide) für einen Kanal

Einschränkungen

Es gelten die folgenden Annahmen und Einschränkungen.

  • Ihre Videos müssen mithilfe eines Dynamic Delivery-Profils aufgenommen oder neu transkodiert werden und über eine MP4-Wiedergabe verfügen
  • Die maximale Eingangsbitrate und Auflösung, die für Cloud Playout unterstützt wird, beträgt 20 MBit / s / 1080p. Dies gilt sowohl für VoD-Inhalte als auch für Live-Eingaben.
  • Brightcove empfiehlt die Verwendung von Videos mit einer Dauer von mindestens 2 Minuten, um einen korrekten Übergang zwischen Videos und der Genauigkeit der elektronischen Programmführung sicherzustellen
  • Derzeit zu Videos hinzugefügte Untertitel (Textspuren) werden in den Cloud Playout-Kanälen nicht angezeigt. Unterstützung für Untertitel wird in einer zukünftigen Version hinzugefügt.
  • Die Mindestdauer für Live-Events beträgt 5 Minuten.
  • Jedes Live-Ereignis in Cloud Playout teilt eine 1: 1-Beziehung mit einem Video Cloud-Remote-Asset. Dieses Remote-Asset kann nicht in anderen Kanälen wiederverwendet werden. Wenn das Live-Ereignis aus dem Kanal entfernt wird, wird das zugehörige Remote-Asset automatisch gelöscht. Wenn ein Kanal gelöscht wird, werden auch alle zugehörigen Remote-Assets automatisch gelöscht.
  • Videos geteilt Zu dem Konto eines anderen Kontos kann derzeit kein Cloud Playout-Kanal hinzugefügt werden.
  • Wenn gerade ein Live-Event abgespielt wird, aber kein aktiver Feed verfügbar ist, wird der Slate-Inhalt in einer Schleife abgespielt, bis der Live-Feed fortgesetzt wird oder das Event endet. Es dauert ungefähr 30 Sekunden, bis der Slate MP4 abgespielt wird.
  • Wenn das Video Cloud-Konto für die Geobeschränkung konfiguriert ist, wird der Kanal angezeigt Ausgabebereich sollte in einer Region erstellt werden, die mit der Geobeschränkungsrichtlinie von Video Cloud kompatibel ist.
  • Für die beste Erfahrung empfiehlt Brightcove die Verwendung des Google Chrome-Browsers.
