Das Campaign-Modul ermöglicht es Video Cloud, Anzeigedaten von Brightcove-Playern mit Oracle Eloqua zu synchronisieren. In Oracle Eloqua können diese Daten für Segmentierung, Kampagnenlogik, Berichte, Lead-Scoring und Personalisierung der Kommunikation verwendet werden. Die Integration des Campaign-Moduls mit Oracle Eloqua besteht aus zwei Hauptkomponenten:

  • Ein Brightcove Player-Plug-In zum Verfolgen von Videoansichtsdaten
  • Ein Synchronisierungsprozess zum Senden von Tracking-Daten aus der Video Cloud an Oracle Eloqua

Bedarf

Um das Campaign-Modul mit Oracle Eloqua zu verwenden, müssen Sie:

  • Haben Sie ein professionelles oder Enterprise Video Cloud-Konto
  • Aktivieren Sie das Campaign-Modul für Ihr Konto, indem Sie die erforderlichen Informationen im Begrüßungsbildschirm eingeben, der beim Öffnen des Campaign-Moduls angezeigt wird
  • Klicken Sie sich bei der ersten Verwendung des Moduls durch den Servicevertrag
  • Verwenden Sie beim Veröffentlichen Ihres Players den Veröffentlichungscode "Advanced Embed" (In-Page). Der Veröffentlichungscode von Standard Embed (iframe) funktioniert nicht, da das Plugin in einem iframe ausgeführt wird und daher nicht auf URL-Parameter der übergeordneten Seite zugreifen kann und möglicherweise nicht die richtigen Cookies erhält
  • Halten Sie Oracle Eloqua 10 und Anmeldeinformationen bereit, um sich anzumelden: https://login.eloqua.com
    Notiz: Diese Anmeldeinformationen werden auf einer Oracle Eloqua-Anmeldeseite eingegeben und nicht direkt an Brightcove weitergegeben.
  • Der Berechtigungsnachweis, mit dem Sie sich bei Oracle Eloqua anmelden, sollte dem hinzugefügt werden Fortgeschrittene Benutzer - Marketing gruppieren und haben die folgenden Berechtigungen:
    • API verbrauchen
    • Installieren Sie Cloud-Anwendungen
    • Einrichten von Marketingkampagnen

Eigenschaften

Einige der Hauptfunktionen der Integration von Video Cloud und Oracle Eloqua sind:

  • Bleiformulare - Erstellen Sie Eloqua-Formulare, die automatisch angezeigt werden, wenn Zuschauer Videos ansehen. Weitere Informationen finden Sie unter Erstellen von benutzerdefinierten Lead-Formularen für Oracle Eloqua.
  • Datei Format - Videowolken-Anzeigedaten werden an ein benutzerdefiniertes Datenobjekt (CDO) in Oracle Eloqua gesendet. Die Daten, die auf Oracle Eloqua repliziert werden, lauten wie folgt:
    • Videoname
    • Video ID
    • Konto-ID (Video Cloud)
    • Seiten-URL (URL der verweisenden Seite)
    • Player ID (Video Cloud Player)
    • % Schaute
  • Datentransfer - Das Anzeigen von Daten wird stündlich synchronisiert und es kann 2-4 Stunden dauern, bis sie in Oracle Eloqua angezeigt werden.
  • Vorratsdatenspeicherung - Die gesammelten Daten können optional bis zu 1 Monat in der Video Cloud gespeichert werden.

Verbindung herstellen

Bevor Sie die Integration mit Eloqua konfigurieren, empfehlen wir Ihnen, ein neues Browserfenster zu öffnen und sich manuell von Ihrem Eloqua-Konto abzumelden. Dadurch wird sichergestellt, dass Sie bei der Konfiguration der Integration aufgefordert werden, sich bei Ihrem Eloqua-Konto anzumelden, und Sie können bestätigen, dass Sie das richtige Konto verwenden.

Führen Sie die folgenden Schritte aus, um die Verbindung zwischen Video Cloud und Oracle Eloqua herzustellen.

  1. Öffnen Sie das Kampagnenmodul und klicken Sie auf Datenverbindungen in der linken Navigation.
  2. Klicken Neue Verbindung hinzufügen.
  3. Wählen Eloqua als die Integration.
    Hinweis: Die verfügbaren Datenverbindungen basieren auf Ihrem Video Cloud-Abonnement.
  4. (Optional) Geben Sie a ein Verbindungsname.
  5. Klicken Verbinden.
  6. Klicken Stellen Sie eine Verbindung zu Eloqua her.
  7. Sie werden aufgefordert, sich bei Eloqua anzumelden. Geben Sie Ihren Eloqua Firmennamen, Benutzernamen und Ihr Passwort ein. Klicken Einloggen.
  8. Wenn die Anmeldung erfolgreich ist, werden Sie aufgefordert, das zu installieren Brightcove Video Connect für Eloqua App. Klicken Akzeptieren und installieren.
  9. Sie werden aufgefordert, Eloqua für das Konto zu aktivieren. Klicken Aktivieren.
  10. Sie werden aufgefordert, dies zuzulassen Brightcove Video Connect für Eloqua um auf Ihr Eloqua-Konto zuzugreifen. Klicken Einloggen.
  11. Bestätigen Sie Ihren Firmennamen und Benutzernamen und klicken Sie auf Akzeptieren.
  12. Sie werden aufgefordert, Eloqua einzurichten. Klicken Richten Sie Eloqua ein.

Integration konfigurieren

Sobald die Verbindung zwischen Video Cloud und Oracle Eloqua hergestellt wurde, können Sie die Integration konfigurieren.

  1. Prüfen Synchronisieren Sie Videoansichten mit Eloqua als externe Aktivitäten Ansichten als externe Aktivitäten synchronisieren. Externe Aktivitäten sind ein weiterer Ort, um Informationen in Eloqua zu platzieren. Verlage können sich dafür entscheiden, Berichte zu erstellen oder Dinge in Eloqua mithilfe externer Aktivitäten auszulösen. Weitere Informationen finden Sie in der Eloqua Dokumentation.
  2. Standardmäßig ist die Verbindungsstatus wird sein aktiviert. Das Verbindungsstatus Legt fest, ob Video Cloud-Video-Engagement-Daten an Oracle Eloqua gesendet werden.
    • Aktiviert - Daten werden gesammelt und an Eloqua gesendet
    • Deaktiviert - Es werden keine Daten gesammelt (Integration ist deaktiviert)
  3. Standardmäßig, Verfolgen Sie Daten für unbekannte Zuschauer wird sein Deaktiviert. Diese Einstellung bestimmt, ob Video-Engagement-Daten in Video Cloud gespeichert werden, wenn der Viewer unbekannt ist. Nur Daten für bekannte Zuschauer werden an Eloqua gesendet.
    • Aktiviert - Daten für alle Zuschauer werden in Video Cloud gesammelt und gespeichert
    • Deaktiviert - Daten für unbekannte Zuschauer werden nicht in der Video Cloud gespeichert
  4. Wählen Sie eine Datenaufbewahrungsrichtlinie aus. Die Datenaufbewahrungsrichtlinie legt fest, wie lange die Anzeigedaten und Lead-Formulardaten (bei Verwendung eines Campaign-Lead-Formulars) in Ihrem Video Cloud-Konto aufbewahrt werden. Durch das Speichern von Daten in Video Cloud können Sie unterbrochene Synchronisierungen wiederherstellen. Außerdem versucht Campaign weiterhin Ereignisse, die nicht synchronisiert werden können (normalerweise weil der Zuschauer zum Zeitpunkt der Videowiedergabe kein bekannter Kontakt ist), bis das Ereignis den Zeitraum der Datenaufbewahrung überschritten hat. Sie sollten die Aufbewahrungseinstellungen gemäß Ihrer Datenschutzrichtlinie anpassen - Es ist nicht erforderlich, Benutzerdaten in der Video Cloud zu speichern. Daten werden nach dem Aufbewahrungszeitraum gelöscht.

Klicken Sie auf, um die Integration mit Oracle Eloqua zu entfernen Trennen. Unten auf der Seite werden alle für die Kampagne aktivierten Spieler und die zugehörigen Lead-Formulare aufgelistet.

Das Campaign-Modul unterstützt mehrere Verbindungen zu Marketingplattformen. Klicken Neue Verbindung hinzufügen um zusätzliche Integrationen zu konfigurieren.

Überprüfung der Integration

Wenn die Integration konfiguriert ist, testen Sie die Integration, um zu bestätigen, dass Videoansichtsdaten an Eloqua gesendet werden. Informationen zum Überprüfen der Integration finden Sie in der Eloqua Integrationstesthandbuch.

Erstellen eines kampagnenfähigen Players

Video-Engagement-Daten werden nur dann an Eloqua gesendet, wenn Sie einen kampagnenfähigen Videoplayer verwenden. Informationen zum Erstellen eines kampagnenfähigen Brightcove-Players finden Sie unter Erstellen eines kampagnenfähigen Players.

Übergabe einer Tracking-ID

Das Campaign-Modul unterstützt die Möglichkeit, einen Tracking-ID-Wert an Eloqua zu übergeben. Die Tracking-ID kann beispielsweise zum Verfolgen einer Kampagnen-ID verwendet werden. Die Tracking-ID kann an die URL angehängt werden, die den für die Kampagne aktivierten Player anzeigt, oder dem Player-Einbettungscode hinzugefügt werden.

  • Beispiel: An URL angehängt
  http://www.myurl.com/video.html?bctrackingid = 12345
  • Beispiel: Zum Einbettungscode des Players hinzugefügt
  <video data-video-id="6033402539001" data-account="1486906377" data-player="default" data-embed="default" data-application-id class="video-js" data-bc-trackingid="12345" Kontrollen>< /Video>< script src="//players.brightcove.net/1486906377/default_default/index.min.js">< /Skript>

Campaign leitet den Wert dann als weiteres synchronisiertes Feld an Eloqua weiter.

Betrachter identifizieren

Kampagnenaktivierte Spieler mit einer Eloqua-Integration rufen die APIs von Eloqua auf, um die Besucher-ID zu erhalten, die häufig als Eloqua-GUID bezeichnet wird. Diese ID wird in den Anzeigedaten des Campaign-Moduls erfasst und verwendet, um Anzeigedaten mit Eloqua zu synchronisieren.

Eloqua verwendet seinen eigenen Satz von Cookies. Wenn ein Brightcove Player mit einer Eloqua-Integration konfiguriert ist, erfasst Campaign die GUID im Eloqua-Cookie und kommuniziert mit Eloqua serverseitig, um herauszufinden, wer der Benutzer ist.

Die E-Mail-Adresse des Benutzers kann auch übergeben und zur Identifizierung des Betrachters verwendet werden. Weitere Informationen und ein Beispiel finden Sie unter So identifiziert das Campaign-Modul Zuschauer.
