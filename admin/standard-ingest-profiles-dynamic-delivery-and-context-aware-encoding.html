Einführung

Beim Erstellen einer Dynamic Ingest-Anfrage mit dem Dynamische Aufnahme-API oder beim Hochladen von Videos mit dem Upload-Modul müssen Sie ein Profil angeben, das die Anzahl und den Typ der Wiedergabeversionen bestimmt, die durch den Video Cloud-Aufnahmeprozess erstellt werden. Dieses Thema enthält detaillierte Informationen zu den einzelnen Themen Dynamische Lieferung und Kontextbewusste Codierung (CAE) Profile, einschließlich HEVC (h.265) Profile.

Anmerkungen:
  • Context Aware Encoding und HEVC-Codierung sind kostenpflichtige Add-Ons für Video Cloud. Wenn Sie diese Funktionen Ihrem Konto hinzufügen möchten, wenden Sie sich an Ihren Account Manager.

Eine Übersicht über Dynamic Delivery finden Sie unter Übersicht über die dynamische Zustellung.

Eine Übersicht über die kontextbezogene Codierung finden Sie unter Übersicht über die kontextbezogene Codierung.

Eine Übersicht über HEVC finden Sie unter HEVC Video

Anmerkungen:

  • Dieses Dokument zeigt Profile für die reguläre dynamische Bereitstellung und kontextbezogene Codierung, einschließlich HEVC-Profile
  • Aufgrund der unterschiedlichen Funktionsweise von statischen Dynamic Delivery- und Context Aware Encoding-Profilen werden in den folgenden Profildetails unterschiedliche Informationen angezeigt:
    • Bei statischen Dynamic Delivery-Profilen wird eine Liste der enthaltenen Wiedergaben angezeigt
    • Für kontextbewusste Codierungsprofile wird eine Tabelle mit dynamischen Profileinstellungen angezeigt, anhand derer bestimmt wird, welche Wiedergaben erstellt werden sollen

Es ist auch möglich, eigene benutzerdefinierte Aufnahmeprofile zu erstellen. Informationen finden Sie im entsprechenden Thema:

  • Erstellen von Aufnahmeprofilen für die dynamische Zustellung
  • Erstellen von Aufnahmeprofilen für die kontextbezogene Codierung

Wiedergaben

Einzelheiten zu allen verfügbaren Standardwiedergaben, die Sie in Ihren Aufnahmeprofilen verwenden können, finden Sie in den folgenden Tabellen.

Anmerkungen:
  1. Dies sind alle Wiedergaben, die zur Verwendung in Dynamic Delivery-Profilen verfügbar sind. Einige von ihnen werden derzeit möglicherweise nicht in den Standardprofilen verwendet.
  2. Die Fragmentdauer und Segmentlänge, die Sie in der Tabelle sehen, spiegeln nicht die tatsächlich gelieferten Segmentlängen wider. Dynamic Delivery kann abhängig von den Bedingungen für eine bestimmte Wiedergabesitzung Segmente unterschiedlicher Länge liefern. Standardmäßig werden 10-Sekunden-Segmente für HLS und 6-Sekunden-Segmente für DASH geliefert. Sieh dir das an Hinweis unten für eine ausführlichere Erklärung von Fragmenten und Segmenten.

Anmerkungen
  • [1-1] Wenn Sie in einem Aufnahmeprofil auf eine Wiedergabe verweisen, Stellen Sie sicher, dass Sie dem Namen voranstellen default/ - zum Beispiel: default/audio128
  • [1-2] Ein Segment entspricht einer herunterladbaren Datei. Mit HLS können Sie die Datei beispielsweise alle 6 Sekunden segmentieren. Jedes Segment enthält genügend Informationen, damit es unabhängig von anderen Segmenten dekodiert und abgespielt werden kann. Ein HLS-Manifest listet jeden Segmentpfad und seine Dauer auf.

    Ein Fragment bezieht sich auf Fragmente einer MP4-Datei. Ein nicht fragmentierter MP4 enthält Metadaten am Anfang oder Ende der Datei und erfordert, dass die Daten (FTYP, MOOV Atome) vor dem Medium (MDAT Atom). Um das Streaming zu vereinfachen, kann eine MP4-Datei fragmentiert werden, sodass die für die Wiedergabe erforderlichen Metadaten sowie die Medien in Fragmente innerhalb der Datei aufgeteilt werden. Dies ermöglicht es den Spielern, leicht nach bestimmten Medienblöcken zu suchen und reibungslose Übergänge zwischen verschiedenen Wiedergaben zu ermöglichen. Die Fragmentierung wurde ursprünglich für DASH erstellt, wird jetzt aber auch von HLS unterstützt.

    Sie können Medien auch so fragmentieren und streichen, dass ein Segment (eine Datei) mehrere Fragmente enthalten kann. Es ist üblich, die Segment- und Fragmentdauer gleich zu machen. Beachten Sie, dass duration und length Beziehen Sie sich bei der Erörterung von Fragmenten und Segmenten auf dasselbe.
