Mit dem Tool für die gerichtete Migration können Sie festlegen, dass ein Brightcove-Player anstelle eines Smart Players geladen werden soll. Das Tool bietet eine einfache Möglichkeit, Videos in Brightcove-Playern zu veröffentlichen, anstatt ältere Smart Player zu verwenden. Es sind keine Änderungen an Ihrem vorhandenen Veröffentlichungscode erforderlich.

Dieses Thema enthält die folgenden Abschnitte:

  • Gezielte Migrationsanforderungen
  • Mapping erstellen
  • Testen von Spielerzuordnungen
  • Bearbeiten von Player-Zuordnungen
  • Spielerzuordnungen löschen
Hinweis: Die Smart Player to Brightcove Player Mappings werden nach dem Lebensende des Smart Players Ende 2017 weiterhin funktionieren.

Anforderungen an die gerichtete Migration

Die folgenden Anforderungen gelten für die Verwendung des Directed Migration Tool:

  • Die Benutzeroberfläche für gerichtete Migration wird nur angezeigt, wenn Sie 3 oder mehr Smart Player in Ihrem Konto haben
  • Sie sollten einen neuen Brightcove-Player erstellen, der über die mit dem Brightcove-Player kompatiblen Stile, Anpassungen und Plugins verfügt
  • Das Directed Migration Tool wird für Spieler mit Wiedergabelisten nicht unterstützt.
  • Der Smart Player-API-Code auf Ihrer vorhandenen Seite funktioniert nach dem Ersetzen des Players nicht mehr.

Einschränkungen

Beachten Sie die folgenden Einschränkungen des Directed Migration Tool:

  • Die gerichtete Migration funktioniert bei Verwendung des Smart Player-Einbettungs-Veröffentlichungscodes oder mit verkürzten oder benutzerdefinierten URLs (bcove.me oder link.brightcove.com).
  • Die gerichtete Migration funktioniert nicht mit dem Nur-Flash-Einbettungscode aus Legacy Studio.

Mapping erstellen

Zuordnungen werden verwendet, um festzulegen, dass ein Smart Player durch einen Brightcove Player ersetzt werden soll. Gehen Sie folgendermaßen vor, um eine neue Zuordnung zu erstellen:

  1. Drücke den ADMINISTRATOR Link im Studio-Header.
  2. Drücke den Spielereinstellungen Verknüpfung.
  3. Klicken Sie auf Mapping erstellen. Eine Liste der Smart Player im Konto wird angezeigt.
  4. Wählen Sie einen oder mehrere Smart Player aus, die durch einen Brightcove-Player ersetzt werden sollen. Klicken Sie auf Weiter.
  5. Wählen Sie einen Brightcove-Player aus, der anstelle der Smart Player verwendet werden soll. Wenn ein Brightcove-Player ausgewählt ist, wird eine Vorschau des Players angezeigt.
  6. Klicken Sie auf Weiter. Anweisungen für Testen von Player-Zuordnungen wird Angezeigt werden.

    Hinweis: Es kann bis zu 5 Minuten dauern, bis Spielerzuordnungen wirksam werden.

  7. Klicken Sie auf Speichern. Eine Liste der aktuellen Zuordnungen wird angezeigt.

Nach dem Erstellen eines Mappings lautet der Status Testen. Sobald das Mapping getestet und festgeschrieben wurde, wird der Status Commited sein. Zuordnungen mit dem Status Engagiert sein rendert einen Brightcove-Player anstelle des Smart Players.

Testen von Spielerzuordnungen

Sie sollten alle Spielerzuordnungen testen, die erstellt wurden, bevor sie festgeschrieben werden. Gehen Sie folgendermaßen vor, um eine Player-Zuordnung zu testen:

  1. Suchen Sie eine Seite, auf der Ihr älterer Smart Player angezeigt wird.
  2. Ergänzen Sie die enableMapping=true Parameter zur Seiten-URL.
    • Wenn die Seiten-URL keine URL-Parameter enthält, fügen Sie hinzu ?enableMapping=true bis zum Ende der URL. Beispielsweise:
      http://www.mysite.com/videos.html?enableMapping=true
    • Wenn die Seiten-URL URL-Parameter enthält, fügen Sie hinzu &enableMapping=true bis zum Ende der URL. Beispielsweise:
      http://www.mysite.com/videos.html?myparams=true&enableMapping=true

    Der neu zugeordnete Brightcove-Player sollte angezeigt und das Video abgespielt werden.

    Wenn Sie mit der Funktionsweise des Players zufrieden sind, kehren Sie zu Video Cloud Studio zurück.

  3. Aktivieren Sie das Kontrollkästchen neben der Player-Zuordnung und klicken Sie auf Zuordnung festschreiben um den neuen Brightcove Player für alle Website-Besucher sichtbar zu machen.

Bearbeiten von Player-Zuordnungen

Gehen Sie folgendermaßen vor, um eine Player-Zuordnung zu bearbeiten:

  1. Drücke den ADMINISTRATOR Link im Studio-Header.
  2. Drücke den Spielereinstellungen Verknüpfung. Eine Liste der Spielerzuordnungen wird angezeigt.
  3. Aktivieren Sie das Kontrollkästchen neben der zu bearbeitenden Player-Zuordnung. Klicken Sie auf Mapping bearbeiten.
  4. Wählen Sie einen neuen Brightcove-Player aus, dem Sie eine Zuordnung zuweisen möchten. Klicken Nächster und dann speichern um die Änderungen zu speichern.

Stellen Sie sicher, dass Sie alle Änderungen an Spielerzuordnungen testen.

Spielerzuordnungen löschen

Gehen Sie folgendermaßen vor, um eine Player-Zuordnung zu löschen:

  1. Drücke den ADMINISTRATOR Link im Studio-Header.
  2. Drücke den Spielereinstellungen Verknüpfung. Eine Liste der Spielerzuordnungen wird angezeigt.
  3. Aktivieren Sie das Kontrollkästchen neben der zu löschenden Player-Zuordnung. Klicken Sie auf Mapping löschen.
  4. Bestätigen Sie das Löschen, indem Sie auf Löschen klicken.
