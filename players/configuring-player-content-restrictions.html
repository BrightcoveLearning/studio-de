Der Spieler Inhaltsbeschränkung Eigenschaften können verwendet werden, um einen Spieler auf bestimmte Länder zu beschränken oder einen Spieler aus bestimmten Ländern zu blockieren. Einem Spieler können Domäneneinschränkungen hinzugefügt werden, um die Domänen einzuschränken, in denen der Spieler verwendet werden kann.

Anmerkungen:

  • Das Geo-Einschränkung Eigenschaften sind nur für Enterprise- und Professional-Konten verfügbar. Domain-Einschränkung ist für alle Konten verfügbar.
  • Die IP-Adresse kann auch verwendet werden, um die Wiedergabe einzuschränken. Weitere Informationen finden Sie unter Einschränken der Videowiedergabe mithilfe der IP-Adresse.
  • IE 10 und 11 zeigen im Kompatibilitätsmodus Fehler bei der Domäneneinschränkung an, wenn Inhalte mit Domäneneinschränkung angezeigt werden.

Anzeigen und Bearbeiten der Inhaltsbeschränkung Eigenschaften für einen Spieler, öffnen Sie das Modul Spieler, klicken Sie auf einen Spielernamen und klicken Sie auf Inhaltsbeschränkung im linken Navigationsmenü.

Hinweis: Alle an einem Spieler vorgenommenen Änderungen können bis zu 5 Minuten dauern, bis sie bei veröffentlichten Spielern angezeigt werden.

Domäneneinschränkungen konfigurieren

Für Spieler können Domänenbeschränkungen festgelegt werden, um die Domänen einzuschränken, in denen Spieler verwendet werden können. Wenn der Player-Veröffentlichungscode kopiert und auf einer anderen Site verwendet wird, verhindern Domain-Einschränkungen, dass der Player Videos lädt. Publisher müssen den In-Page-Einbettungscode verwenden, damit die Wiedergabe in den Domänen funktioniert, für die eine Whitelist erstellt wurde. Publisher können den iframe-Einbettungscode nicht für eine Whitelist-Domain (oder eine andere Domain) verwenden und erwarten, dass die Wiedergabe ordnungsgemäß funktioniert.

Anmerkungen:

  • Während Domain-Einschränkungen dazu beitragen können, die unbefugte Verbreitung Ihrer Inhalte zu verhindern, sollten Kunden, die maximale Inhaltssicherheit suchen, stattdessen DRM-Lösungen in Betracht ziehen. Sehen Digital Rights Management (DRM) in der Video Cloud für mehr Informationen.
  • Für Betrachter, die Internet Explorer verwenden, ist für die Domäneneinschränkungsfunktion Version 10 oder höher erforderlich. Zuschauer, die Internet Explorer <10 verwenden, erhalten überhaupt keine Wiedergabe.
  • Domäneneinschränkungen werden unterstützt, wenn Sie eine Galerie-In-Page-Erfahrung und den empfohlenen Einbettungscode (JavaScript) verwenden.
  • Domain-beschränkte Player sind nicht mit AMP-Integrationen kompatibel. Dies liegt daran, dass durch die AMP-Integration ein Iframe erstellt wird, der wiederum nicht von domänenbeschränkten Spielern unterstützt wird.
  • Bei Verwendung der Domaineinschränkungsfunktion sollten Domaineinschränkungen für alle Spieler in Ihrem Konto konfiguriert werden. Wenn nicht alle Player mit Domäneneinschränkungen konfiguriert sind, kann jemand den Richtlinienschlüssel von einem ungeschützten Player nehmen und zum Abspielen von Videos verwenden.
  • Während die Anzahl der Domänen, die in Studio hinzugefügt werden können, unbegrenzt ist, ist die Länge der URL für die API-Anforderung begrenzt. In dieser URL ist die durch Kommas getrennte Liste der Domänen enthalten. Daher ist die Anzahl der Domänen, die angegeben werden können, begrenzt. Die Anzahl hängt jedoch davon ab, wie lang die Domänennamen sind.

Gehen Sie folgendermaßen vor, um eine Domänenbeschränkung für einen Player zu konfigurieren:

  1. Geben Sie eine oder mehrere vollständige Domain-URLs ein (z. B. http://www.brightcove.com oder https://www.brightcove.com), um die Verfügbarkeit Ihres Players auf diese bestimmten Domains zu beschränken. Stellen Sie sicher, dass Sie entweder http: // oder https: // angeben. Klicken +.

    Anmerkungen:

    • Es müssen vollständige URLs angegeben werden. Platzhalter in URLs werden nicht unterstützt. URLs sollten nicht mit einem / enden.
    • Es werden nur Domänen und keine bestimmten Pfade unterstützt. Zum Beispiel können Sie hinzufügen https://www.brightcove.com, aber nicht https://www.brightcove.com/documentation
  2. Klicken speichern um die Änderungen zu speichern.

Geofilter konfigurieren

Mithilfe der Geofilterung können Sie steuern, welche Länder einen Player verwenden dürfen und welche nicht.

Anmerkungen:

  • Geofilterung ist eine Funktion, die Enterprise- und Professional-Konten gegen eine zusätzliche Gebühr zur Verfügung steht. Wenden Sie sich an Brightcove, wenn Sie Ihr Konto um eine Geofilterung erweitern möchten.
  • Geofilterung kann auch sein auf einzelne Videos angewendet. Video Cloud wendet sowohl Player- als auch Video-Geofilterungsrichtlinien an. Wenn ein Land von einem der beiden Länder ausgeschlossen wird, ist das Video dort nicht verfügbar.

So legen Sie die Geofiltereigenschaften für einen Player fest:

  1. Wählen Ermöglichen Sie die Anzeige aus den folgenden Ländern oder Verhindern Sie die Anzeige aus den folgenden Ländern.
  2. Geben Sie die entsprechenden Ländernamen ein. Während der Eingabe wird die Länderliste gefiltert.
  3. Klicken Sie auf ein Land und drücken Sie die Eingabetaste, um es der Liste hinzuzufügen.
  4. Klicken speichern.
